R_Practice

This simple exercise is designed to get you started with Rmarkdown and DNA methylation microarray data analysis. You can view the results here.

Prerequisites

You will need git and git-lfs installed to clone the repository.

Once you have them installed, type

git config --global user.name "your_username"

and

git config --global user.email "your_email"

to set Git up. Type the same commands without the last part (the text in “”) to make sure you set it correctly.

You will most likely need to install some R packages, but this part is up to you to figure out yourself :) .

You will also need rmarkdown and pandoc.

I suggest using the development version of rmarkdown, for that you will need devtools. In R, type

install.packages("devtools")
library(devtools)
install_github("rstudio/rmarkdown")

Geting the repository

Once you have everything set up, use

git clone git@github.com:algisk/R_Practice.git

if you have ssh set up or

git clone https://github.com/algisk/R_Practice.git

if you want to use the HTTPS version (not recommended).

This will create a folder caled “R_Practice” in your current directory.

Working on the exercises

Once in the repository, create a new branch

git checkout -b branch_name

You can make whatever you want in this branch and once you make some changes that you are happy with, add any untracked files, commit with an informative message and push to the remote repository

git status

If the files you were working on are in red, type

git add filename

If everything you need comes up green once you check status, commit the changes

git commit -m "very informative message, super useful"

Finally, push everything to your new branch

git push origin branch_name

Rendering the final result

When you feel like your final result is good to go, add the .Rmd file to _site.yml in the same format. Let’s say you created results.Rmd, in the .yml it would look like

    - text: "Result"
      href: result.html

Keep in mind that indentation is important!

Finally, open R and type

rmarkdown::render_site()

from the root project directory.

If you completed everything correctly, you should end up with a folder named _site with a bunch of files. Open any of the htmls in the folder and your new results should show up as a separate tab.

Running make

Running make in the project root directory will generate a folder named docs which is useful if you want to publish the results through Github Pages.

If you are working on the master branch, you can also run make publish to automaticaly push the updated version of the website to algisk.github.io/R_Practice.
